# 品質 Checklist（對齊 Gate G0-G6）

> **版本**: v2.0
> **最後更新**: 2026-03-22

---

## 概述

本文件定義每個 Gate 的品質檢查項目，與 `gate-keeper.ts` 的 `GATE_CHECKLISTS` 對齊。Gate 審核時依據此 Checklist 逐項確認。

## G0 — 需求確認

- [ ] 需求文件已建立（包含功能範圍、用戶角色、核心 User Story）
- [ ] 用戶故事已定義（每個故事有角色、目標、驗收條件）
- [ ] 驗收標準已明確（量化指標或可驗證的完成條件）
- [ ] 技術可行性已確認（關鍵技術選型已 PoC 驗證，風險項已列出）

**通過標準**：專案需求清晰、可執行，技術上可行。

## G1 — 圖稿審核

- [ ] UI/UX 圖稿已完成（所有頁面的 Wireframe/Mockup 已產出，互動流程已標註）
- [ ] 架構設計已審核（系統架構圖已繪製，模組職責與依賴關係已明確）
- [ ] API 規格已定義（所有端點的 Request/Response Schema 已文件化）
- [ ] 資料模型已確認（DB Schema 已定義，包含欄位型別、索引、關聯與 Migration 計畫）

**通過標準**：設計方案完整，團隊對實作路徑有共識。

## G2 — 程式碼審查

### CRITICAL（安全性，必須全過）

- [ ] SQL 注入防護：所有 DB 查詢使用參數化，無字串拼接
- [ ] Race Condition：共用資源存取有適當鎖定或序列化機制
- [ ] LLM 信任邊界：AI 輸出不直接用於 SQL/shell/eval，有 sanitize
- [ ] 認證授權：所有端點有適當的認證和權限檢查

### INFO（品質，建議全過）

- [ ] 功能實作已完成：所有 Sprint 任務狀態為 done
- [ ] Magic Number：數值常數已提取為命名常數或設定
- [ ] Dead Code：無未使用的函數、變數、import
- [ ] 文件已同步：README、API 文件、架構圖已同步更新

**通過標準**：CRITICAL 全過 + INFO 無重大遺漏。

## G3 — 測試驗收

### T1（編譯層，必須全過）

- [ ] TypeScript 編譯通過：`tsc --noEmit` 無錯誤
- [ ] Lint 通過：ESLint + Prettier 無錯誤或警告

### T2（測試層，必須全過）

- [ ] 單元測試通過：所有單元測試 Pass，核心模組覆蓋率 ≥ 80%
- [ ] E2E 測試通過：Playwright E2E 測試全部 Pass

### T3（驗證層，盡量全過）

- [ ] 效能檢查：關鍵操作回應時間 < 目標值，無明顯效能退化
- [ ] 跨瀏覽器/平台：目標平台手動驗證無重大差異
- [ ] 無重大 Bug：無 Critical/High 等級的未解決 Bug

**通過標準**：T1 + T2 全過，T3 無 Critical。

## G4 — 文件審查

- [ ] 文件更新已完成：CLAUDE.md、README、API 文件、架構圖已同步更新
- [ ] 開發紀錄已補齊：包含變更背景、修改範圍、設計決策
- [ ] 規格書已同步：IPC 介面規格書、產品規格書、資料庫設計已反映最新變更
- [ ] 測試文件已更新：功能測試清單、E2E 測試個案說明已涵蓋新功能

**通過標準**：規範文件與程式碼完全一致，索引已更新。

## G5 — 部署就緒

- [ ] CI/CD 管線通過：Build → Test → Deploy 管線完整執行成功
- [ ] 環境配置已就緒：目標環境的環境變數、密鑰、DNS 已設定
- [ ] 回滾計畫已準備：回滾步驟已文件化，包含 DB Migration 回退方案
- [ ] 監控已設定：錯誤追蹤、效能監控、告警通知已配置

**通過標準**：部署流程已驗證，出問題可快速回滾。

## G6 — 正式發佈

- [ ] 安全審核通過：無 OWASP Top 10 漏洞，依賴項無已知高風險 CVE
- [ ] 上線前最終確認：產品負責人在 Staging 環境完成 UAT 驗收
- [ ] 用戶文件已完成：使用說明、FAQ、Changelog 已撰寫
- [ ] 正式核准發佈：所有利害關係人已簽核，發佈時間與通知計畫已確認

**通過標準**：老闆最終確認，核准上線。

## Sprint 類型對應

| Sprint 類型 | 需要通過的 Gate |
|-------------|----------------|
| full | G0 → G1 → G2 → G3 → G4 → G5 → G6 |
| feature | G0 → G2 → G3 → G4 |
| bugfix | G0 → G2 → G3 |
| release | G0 → G5 → G6 |

> 實際走哪些 Gate 由 G0 審核時動態勾選決定（SOP v4.0）。
