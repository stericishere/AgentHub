# 知識回饋 SOP

> **版本**: v1.0
> **最後更新**: 2026-03-03

---

## 概述

專案完成後（G5 通過），PM 與老闆回顧本專案的經驗，將有價值的技術實踐和流程改善寫回知識庫，形成持續學習循環。

## 流程

```
知識庫模板 → PM 參考 → 專案規範 → 開發 → 完成(G5) → PM 回顧 → 更新知識庫
```

### 第一步：PM 發起回顧

專案通過 G5 後，PM 啟動回顧 Session：
- 整理本專案中用到的技術決策
- 收集各 Agent 的 Session 摘要
- 識別值得沉澱的最佳實踐

### 第二步：與老闆討論

PM 向老闆報告以下內容：
1. **技術收穫**：哪些技術方案特別有效
2. **流程改善**：哪些流程可以優化
3. **踩過的坑**：遇到的問題和解決方式
4. **規範更新建議**：哪些規範需要調整

### 第三步：更新知識庫

老闆審核同意後，PM 執行更新：

| 更新類型 | 位置 | 範例 |
|----------|------|------|
| 通用規範更新 | `knowledge/company/standards/` | 發現新的安全檢查項目 |
| SOP 優化 | `knowledge/company/sop/` | Sprint 流程改善 |
| 模板改善 | `knowledge/company/project-templates/` | 新增好用的架構模式 |
| 新增模板 | `knowledge/company/templates/` | 新的文件模板 |

### 第四步：版本紀錄

更新的文件須更新版本號和日期，並在文件末尾記錄變更原因。

## 知識庫不更新的情況

- 僅適用於特定專案的配置（留在專案 workDir 內）
- 實驗性質的技術選型（等驗證後再考慮）
- 與既有規範衝突的做法（需額外討論）
